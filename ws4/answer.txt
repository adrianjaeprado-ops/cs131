extract ()
- The "for archive in $*; do" loop is what lets the function accept more than one file at a time. so when extracting several archives does not make it run the command over and over.

- The "if [ -f $archive ]" check makes sure the thing being passed in is actually a real existing file before the function tries to do anything with it. If that check fails, the else branch runs instead and just prints an error message reather than letting the script try to extract something that doesn't exist.

- The "case $archive in ... esac" block is a huge part of the function. It looks at the file's extension and matches it against a list of petterns like .zip, .tar.gz, .rar, etc. and each pattern brings the file to the correct extraction tool like unzip for .zi[ or tarxvzf for .tar.gz.

- The final "*)" pattern acts like a catcher for any extensions the function doesn't recognize. Instead of an unoticed failure happening or throwing a system error, it print's a clear message admitting it doesn't know how to handle that file type, which makes the function easier to troubleshoot.

trim ()

- The "local var=$@" takes whatever was passed into the function and stores it in a local variable called var, so the function doesn't accidentally mess with any variables outside of itself.

- The first "var="${var#"${var%%[![:space:]]*}"}"" line is what strips off any leading whitespace. It works by finding the first character in the string that isn't a space and then cutting off everything before it.

- The second "var="${var%"${var##*[![:space:]]}"}"" line does the same thing but from the other direction, removing any trailing whitespace by trimming off everything after the last non-space character.

- The "echo -n "$var"" prints the cleaned-up string without adding a newline at the end. The "-n" matters here because if this function is being used inside another command or being combined with other text, an extra newline could mess up formatting or cause unexpected line breaks.
